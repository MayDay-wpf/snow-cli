# AGENTS.md Context Inject

Snow CLI can load project instruction files and prepend them to **model-bound user messages** when enabled. This is the built-in AGENTS context inject path.

**Default off (opt-in)**: nothing is injected until you set `contextInject.enabled` to `true` in unified settings.

It is separate from:

- **ROLE.md** — persona / hard rules on the **system prompt** path
- **Hooks `additionalContext`** — dynamic inject from `onSessionStart` / `onUserMessage` / `beforeSubAgentStart`

---

## What gets injected

### Sources (order)

1. **Global**: `~/.snow/AGENTS.md` (or override / fallback in that directory)
2. **Project chain**: from git root → current working directory, **one file per directory**

### Filename priority per directory

1. `AGENTS.override.md`
2. `AGENTS.md`
3. Fallback names from settings (default: `CLAUDE.md`)

Empty files are skipped.

### Not loaded by this path

- `ROLE.md` / `ROLE-<id>.md` — still handled by system prompt helpers
- Hook-generated context — still handled by the hooks pipeline

---

## Where it is applied

```text
User types message
  → UI bubble keeps typed text (typedMessage)
  → Hooks / bash enrichment (optional)
  → prependAgentsContext()  ← AGENTS chain
  → Model-bound user message
```

Key points:

- Inject happens on the **user-message** path, not the system prompt
- The chat bubble still shows what the user typed
- Master switch defaults to off; once enabled, the main chat path uses the `full` profile

Implementation:

- Module: `source/prompt/contextInject/`
- Entry: `prependAgentsContext()` in `source/prompt/contextInject/index.ts`
- Wiring: `source/hooks/conversation/chatLogic/useMessageProcessing.ts`

## ROLE.md vs AGENTS.md

| File                       | Path                 | Purpose                                            |
| -------------------------- | -------------------- | -------------------------------------------------- |
| `ROLE.md` / `ROLE-<id>.md` | System prompt        | Persona, hard rules, optional full system override |
| `AGENTS.md` chain          | User message prepend | Project/global coding instructions for the model   |

ROLE behavior:

- Global + project active ROLE can both be loaded
- Identical bodies are content-deduped (first wins)
- AGENTS bodies that exactly match ROLE fingerprints are dropped to avoid double injection

---

## Budget, profile, and dedupe

Defaults (Codex-like):

| Setting            | Default          |
| ------------------ | ---------------- |
| Enabled            | `false` (opt-in) |
| Budget             | `32000` chars    |
| Compact budget     | `8000` chars     |
| Per-file max       | `16000` chars    |
| Primary filename   | `AGENTS.md`      |
| Fallback filenames | `CLAUDE.md`      |
| Write breadcrumb   | `false`          |

Profiles:

- `full` — use configured budget
- `compact` — cap budget at `compactBudgetChars`
- `off` — disable inject

Dedupe policy:

1. Normalize content (trim, CRLF → LF, collapse blank runs)
2. Fingerprint and keep first occurrence in discovery order
3. Drop AGENTS content already present in active ROLE fingerprints

Optional breadcrumb (debug):

- When `writeBreadcrumb: true`, write `.snow/log/injected-rules.txt`

## Configuration

Settings key: `contextInject` in unified settings (`settings.json`, project or global).

```json
{
	"contextInject": {
		"enabled": true,
		"budgetChars": 32000,
		"profile": "full",
		"fallbackFilenames": ["CLAUDE.md"],
		"writeBreadcrumb": false
	}
}
```

| Field               | Type                         | Description                                |
| ------------------- | ---------------------------- | ------------------------------------------ |
| `enabled`           | boolean                      | Master switch (default `false`, opt-in)    |
| `budgetChars`       | number                       | Total inject budget in characters          |
| `profile`           | `full` \| `compact` \| `off` | Inject profile                             |
| `fallbackFilenames` | string[]                     | Extra names after `AGENTS.md`              |
| `writeBreadcrumb`   | boolean                      | Write inject debug file under `.snow/log/` |

## Recommended layout

```text
~/.snow/AGENTS.md                 # Global assistant instructions
repo/
  AGENTS.md                       # Repo-wide project rules
  packages/app/AGENTS.md          # Package-local rules (when cwd is under it)
  ROLE.md                         # Optional persona (system path)
```

Tips:

- Put durable project conventions in `AGENTS.md`
- Put persona / non-negotiable hard rules in `ROLE.md`
- Use `AGENTS.override.md` only when you need a directory-local override of `AGENTS.md`
- Prefer `/init` to generate or refresh project `AGENTS.md`

## Relation to Hooks additionalContext

Both can prepend model-visible text while keeping the UI bubble clean.

Typical order on send:

1. Hook / bash enrichment (`additionalContext`, etc.)
2. Built-in AGENTS prepend
3. Send to model

Use hooks for **dynamic / session-specific** inject (Trellis, runtime state).
Use AGENTS for **stable repo/global instruction files**.

See also: [Hooks Configuration](./07.Hooks%20Configuration.md).
