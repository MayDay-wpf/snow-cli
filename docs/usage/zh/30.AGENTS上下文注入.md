# AGENTS.md 上下文注入

Snow CLI 会自动加载项目指令文件，并 **prepend 到发给模型的 user message**。这是内置的 AGENTS 上下文注入路径。

它与以下机制相互独立：

- **ROLE.md** —— 走 **system prompt** 的人设 / 硬规则
- **Hooks `additionalContext`** —— 来自 `onSessionStart` / `onUserMessage` / `beforeSubAgentStart` 的动态注入

---

## 会注入什么

### 来源（顺序）

1. **全局**：`~/.snow/AGENTS.md`（或该目录下的 override / fallback）
2. **项目链**：从 git root → 当前工作目录，**每个目录最多一份文件**

### 每个目录的文件优先级

1. `AGENTS.override.md`
2. `AGENTS.md`
3. 配置中的 fallback 名称（默认：`CLAUDE.md`）

空文件会被跳过。

### 本路径不会加载

- `ROLE.md` / `ROLE-<id>.md` —— 仍由 system prompt helper 处理
- Hook 生成的上下文 —— 仍由 hooks 管线处理

---

## 注入发生在哪里

```text
用户输入消息
  → UI 气泡保留原文 (typedMessage)
  → Hooks / bash 增强（可选）
  → prependAgentsContext()  ← AGENTS 链
  → 发给模型的 user message
```

要点：

- 注入发生在 **user-message** 路径，不进 system prompt
- 聊天气泡仍显示用户原文
- 主会话默认使用 `full` profile

实现位置：

- 模块：`source/prompt/contextInject/`
- 入口：`source/prompt/contextInject/index.ts` 的 `prependAgentsContext()`
- 接线：`source/hooks/conversation/chatLogic/useMessageProcessing.ts`

---

## ROLE.md 与 AGENTS.md

| 文件 | 路径 | 用途 |
| ---- | ---- | ---- |
| `ROLE.md` / `ROLE-<id>.md` | System prompt | 人设、硬规则，可选完全覆盖 system prompt |
| `AGENTS.md` 链 | User message prepend | 给模型的项目/全局编码指令 |

ROLE 行为：

- 可同时加载全局 + 项目活跃 ROLE
- 内容完全相同会做 content-level 去重（先出现的保留）
- 若 AGENTS 正文与 ROLE 指纹完全一致，会丢弃该 AGENTS 段，避免双份注入

## 预算、profile 与去重

默认值（接近 Codex 风格）：

| 配置项 | 默认 |
| ------ | ---- |
| 启用 | `true` |
| 预算 | `32000` 字符 |
| compact 预算 | `8000` 字符 |
| 单文件上限 | `16000` 字符 |
| 主文件名 | `AGENTS.md` |
| fallback 文件名 | `CLAUDE.md` |
| 写 breadcrumb | `false` |

Profile：

- `full` —— 使用配置预算
- `compact` —— 预算封顶到 `compactBudgetChars`
- `off` —— 关闭注入

去重策略：

1. 规范化内容（trim、CRLF→LF、压缩空行）
2. 按发现顺序指纹去重（先出现的保留）
3. 丢弃与活跃 ROLE 指纹相同的 AGENTS 内容

可选调试 breadcrumb：

- `writeBreadcrumb: true` 时写入 `.snow/log/injected-rules.txt`

## 配置

设置键：统一设置中的 `contextInject`（`settings.json`，项目或全局）。

```json
{
  "contextInject": {
    "enabled": true,
    "budgetChars": 32000,
    "profile": "full",
    "fallbackFilenames": ["CLAUDE.md"],
    "writeBreadcrumb": false
  }
}
```

| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |
| `enabled` | boolean | 总开关（默认 `true`） |
| `budgetChars` | number | 注入总预算（字符） |
| `profile` | `full` \| `compact` \| `off` | 注入 profile |
| `fallbackFilenames` | string[] | `AGENTS.md` 之后的备选文件名 |
| `writeBreadcrumb` | boolean | 是否写入 `.snow/log/` 调试文件 |

## 推荐目录布局

```text
~/.snow/AGENTS.md                 # 全局助手指令
repo/
  AGENTS.md                       # 仓库级项目规则
  packages/app/AGENTS.md          # 包级规则（cwd 在其下时生效）
  ROLE.md                         # 可选人设（system 路径）
```

建议：

- 稳定的项目约定放 `AGENTS.md`
- 人设 / 不可违背硬规则放 `ROLE.md`
- 仅在需要覆盖目录内 `AGENTS.md` 时使用 `AGENTS.override.md`
- 可用 `/init` 生成或刷新项目 `AGENTS.md`

---

## 与 Hooks additionalContext 的关系

两者都可以在保持 UI 气泡干净的前提下，给模型 prepend 可见上下文。

发送时的典型顺序：

1. Hook / bash 增强（`additionalContext` 等）
2. 内置 AGENTS prepend
3. 发送给模型

动态 / 会话态上下文用 hooks（如 Trellis、运行时状态）。
稳定的仓库/全局指令文件用 AGENTS。

另见：[Hooks 配置](./07.Hooks配置.md)。
